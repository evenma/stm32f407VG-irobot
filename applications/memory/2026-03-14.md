- **接口**: I2C @ 0x1C (PB8/SCL, PB9/SDA)
- **功能**: ASR 语音识别 + TTS 文本合成 + 唤醒词检测
- **工作模式**:
  - MODE_FREE_TRIGGER: 自由触发（单次唤醒，说完即停）

- **核心 API**:
  ```c

  wonder_echo_speak_now("hello", 80);          // 立即播报
  wonder_echo_queue_tts(text, vol);            // 加入队列
  wonder_echo_register_on_wakeup(cb);          // 回调注册
  ```
- **MSH 命令**: `echo status`, `echo test`, `echo vol 80`, `echo mode 1`
- **TTS 队列**: 独立线程管理，防止阻塞主循环

- **MSH 命令**: `motor status`, `motor vel left 100`, `motor pid 2.0 0.1 0.05`

## 🔧 关键技术决策

### 架构设计原则
1. **Hiwonder 模型采纳**: Peripherals/ = Object Drivers, Portings/ = Init/Queue Interfaces
2. **全局配置优先**: 所有 100+ 引脚定义先汇总到 global_conf.h 再实现组件
3. **SPI 资源管理**: OLED+LED共享 SPI2，通过互斥锁 (`oled_spi_lock()/unlock()`) 保护
4. **初始化顺序**: Buzzer→LED→OLED(IMU) →RS485 →CANopen Motors

### 优化策略
- **浮点数格式化**: u8g2 不支持 float → 数据放大 10 倍用整数显示 (pitch_int*10)
- **轮询代替中断**: PC13/PC14外中断已分配给其他组件，改用 polling 检测 SW3/SW4
- **异步 TTS 处理**: WonderEcho 队列机制避免阻塞主循环
- **MSH 调试优先**: 所有模块提供 shell 命令方便现场调试

### 内存占用估算
```
总计 RAM: ~42KB
- LED: 1KB
- Buzzer: 0.5KB
- OLED(u8g2): 15KB
- IMU: 2KB
- Ultrasonic: 5KB
- WonderEcho: 8KB
- CAN Motor: 10KB

总计 Flash: ~300KB
```

## 🚨 问题与解决记录

### SPI 互斥冲突 ✅已解决
**问题**: OLED 和 LED同时使用 SPI2 总线导致显示异常  
**原因**: led_porting.c调用 u8g2 API 但 SPI 被 oled_handle.c独占  
**解决**: 
1. `oled_handle.c` 中移除 `static`关键字，公开 `oled_spi_lock()/unlock()` API
2. `led_porting.c` 改为先获取 SPI 互斥锁再访问显示缓冲区
3. 确保初始化顺序：先 init OLED 再 init LED

### OLED 显示字体兼容 ✅已解决  
**问题**: u8g2 库不直接支持浮点数格式化，角度数据显示乱码  
**解决**: 
- 将 float 数据放大 10 倍转换为 int16_t
- 分离整数和小数部分分别拼接字符串
- 例如：`pitch_last = 45.3°` → `buf = "4.5°"` (453/10, 453%10)

### Modbus CRC16 计算 ✅已验证
**测试用例**: 地址 0x01 + 功能码 0x03 + 寄存器 0x0000  
**正确结果**: CRC = 0x8B7B  
**算法**: 标准 MODBUS 多项式 `0xA001` 反向迭代
